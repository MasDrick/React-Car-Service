Техническое задание (ТЗ) на разработку фронтенд-части системы «Автосервис»

1. Общие сведения

Название проекта: Веб-приложение «Автосервис»
Цель разработки: Создание клиентской части информационной системы для управления услугами и заказами автосервиса.
Платформа: Web (десктоп и мобильная версия)
Технологии: React, Redux Toolkit, React Router, Axios, Vite (или CRA), HTML5, CSS3, TailwindCSS / MUI

2. Назначение и цели системы

Приложение предназначено для:

предоставления пользователю возможности записаться на услугу автосервиса;

просмотра списка услуг и статусов заказов;

для администратора — управления услугами и заказами.

Приложение должно взаимодействовать с REST API, реализованным на стороне сервера (FastAPI).

3. Функциональные требования
   3.1. Основные пользовательские роли

Гость — может просматривать услуги и переходить на страницу авторизации.

Клиент — после входа может создавать заказы и отслеживать их статус.

Администратор — может управлять услугами, пользователями и заказами.

(В MVP можно реализовать только «Клиента» и «Администратора».)

4. Описание интерфейса и страниц
   4.1. Страница авторизации / регистрации (/login, /register)

Поля:

Логин

Пароль

Действия:

Авторизация (POST /auth/login)

Регистрация нового пользователя (POST /auth/register)

После успешного входа — редирект на /services.

UI: простая форма по центру, кнопки «Войти» / «Регистрация», индикация ошибок.

4.2. Главная страница (список услуг) (/services)

Отображается список услуг с названием, ценой и продолжительностью.

Действия:

Клиент: может нажать «Записаться» (открывается модальное окно оформления заказа).

Админ: может добавлять/редактировать/удалять услуги (CRUD).

UI:

Карточки услуг в сетке.

Модальное окно добавления/редактирования услуги.

Для администратора — кнопки «Редактировать», «Удалить», «Добавить услугу».

4.3. Страница заказов (/orders)

Таблица или карточки заказов пользователя.

Для каждого заказа:

Название услуги

Дата записи

Статус: Новый, В работе, Готов, Завершён

Действия:

Клиент: только просмотр.

Админ: может менять статус заказа.

UI:
Таблица или карточки с цветовой индикацией статусов.

4.4. Страница администратора (/admin)

Панель управления, содержащая:

Вкладку «Услуги» — CRUD операций.

Вкладку «Заказы» — изменение статусов заказов.

(Опционально) статистику: количество заказов, выручка, загрузка мастеров.

UI:
Навигация по вкладкам, таблицы с фильтрацией, модальные окна редактирования.

4.5. Страница профиля (/profile)

Отображение информации о пользователе.

Возможность выйти из системы.

5. Навигация и маршрутизация
   Путь Компонент Доступ
   /login LoginPage Все
   /register RegisterPage Все
   /services ServicesPage Клиент, Админ
   /orders OrdersPage Клиент, Админ
   /admin AdminDashboard Только Админ
   /profile ProfilePage Авторизованные

Маршрутизация реализуется через React Router v6.
Для ограничения доступа — компонент PrivateRoute, проверяющий токен и роль пользователя.

6. Состояние приложения (Redux store)

Slices:

authSlice — авторизация, хранение токена, данных пользователя.

servicesSlice — список услуг (CRUD).

ordersSlice — список заказов (по пользователю или всем).

uiSlice (опционально) — состояние модалок, уведомлений и загрузки.

7. Взаимодействие с API (через Axios)
   Метод Эндпоинт Описание
   POST /auth/login Вход пользователя
   POST /auth/register Регистрация
   GET /services Получить список услуг
   POST /services Добавить услугу (админ)
   PUT /services/{id} Обновить услугу
   DELETE /services/{id} Удалить услугу
   GET /orders Получить заказы
   POST /orders Создать заказ
   PUT /orders/{id} Изменить статус заказа
